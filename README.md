# DevOps тестовый проект

Необходимо развернуть простое приложение, показывающее текущую погоду, поместить его в Docker-контейнер и настроить базовый мониторинг. Инструмент для мониторинга можно использовать любой.
Создать лёгкое веб-приложение (например, на Python, Node.js, Go, Java или любом другом языке), которое отображает текущую температуру
Упаковать приложение в Docker-контейнер
Настроить мониторинг: доступности погодного веб-приложения (HTTP-check)
Опционально: мониторинг дополнительных параметров приложения
## Структура проекта

```
.
├── main.go              # Основное приложение Go
├── go.mod               # Зависимости Go
├── Dockerfile           # Docker образ приложения
├── docker-compose.yml   # Оркестрация сервисов
├── prometheus.yml       # Конфигурация Prometheus
└── README.md            # Документация
```

## API Endpoints

- `GET /` - Веб-интерфейс с отображением температуры
- `GET /api/temperature` - REST API для получения температуры в JSON формате
- `GET /health` - Health check endpoint
- `GET /metrics` - Prometheus метрики

### Пример ответа API

```json
{
  "temperature": 15.5,
  "unit": "celsius",
  "timestamp": "2025-01-27T10:30:00Z",
  "source": "weather-api"
}

## Переменные окружения

- `PORT` - Порт для запуска приложения (по умолчанию: 8080)
- `WEATHER_CITY` - Город для получения температуры (по умолчанию: Moscow)
- `WEATHER_API_KEY` - API ключ для OpenWeatherMap (опционально, если не указан - используется демо-режим)

## Мониторинг

### Prometheus метрики
Приложение экспортирует следующие метрики:
- `http_requests_total` - Общее количество HTTP запросов
- `http_request_duration_seconds` - Длительность HTTP запросов
- `current_temperature_celsius` - Текущая температура в градусах Цельсия
### Health Checks
health checks:
- Проверка доступности каждые 10 секунд
- Timeout 5 секунд
- 3 попытки перед пометкой как unhealthy
